# 定位布局

## 水平&垂直居中
### 水平居中
- 内联元素水平居中
```html
<div class="main">
	<span class="inline">Test message.</span>
</div>
```
```css
//方法一：在父元素上设置text-align
.main{
	text-align: center;
}
//方法二：position+margin
.main{
    position: relative;
}
.inline{
    position: absolute;
    left: 50%;
    transform: translateX(-50%);
}
```
- 块级元素水平居中
```html
<div class="main">
	<div class="block">Test message.</div>
</div>
```
```css
//方法一：为块级元素设置width和margin
.block{
	width: 80%;
	margin: auto;
}
//方法二：position+margin
.main{
    position: relative;
}
.inline{
    position: absolute;
    left: 50%;
    transform: translateX(-50%);
}
```
### 垂直居中
- position+transform：内联和块级元素均适用
```html
<div class="main">
	<span class="inline">Test message.</span>
</div>
<div class="main">
	<div class="block">Test message.</div>
</div>
```
```css
.main{
	position: relative;
}
.inline,
.block{
	position: absolute;
	left: 50%;
	top: 50%;
	transform: translate(-50%, -50%);
}
```
- FlexBox：内联和块级元素均适用
```html
<div class="main">
	<span class="inline">Test message.</span>
</div>
<div class="main">
	<div class="block">Test message.</div>
</div>
```
```css
.main{
	display: flex;
    justify-content: center;   //主轴方向
    align-items: center;   //垂轴方向
}
.inline,
.block{
    align-self: center;
}
```
- 视口单位vh/vw：仅在块级元素相对视口垂直居中有效
```html
<div class="main">
	<div class="block">Test message.</div>
</div>
```
```css
.main{
	margin: 50vh auto 0;
    transform: translateY(-50%);
}
```
- table+vertical-align：对内联和块级元素均适用
```html
<div class="main">
	<span class="inline">Test message.</span>
</div>
<div class="main">
	<div class="block">Test message.</div>
</div>
```
```css
.main{
	display: table;
}
.inline,
.block{
	display: table-cell;
	vertical-align: middle;
}
```

## 页面布局方式
### flow布局
>流式布局，按照块级元素从上至下，内联元素从左到右的顺序进行布局，通过百分比来设置元素width。但是，元素height是固定的，在自适应布局时，会导致元素宽高比失衡。

```html
<div class="main">
    <div class="one">This is a test.</div>
</div>
```
```css
.main{
    width: 80%;
    height: 200px;
    margin: auto;
}
```

### rem布局
>rem布局，rem即根元素(html)font-size的大小，利用rem为单位设置元素width/height。在自适应布局时，元素宽高比保持不变。

```html
<div class="main">
    <div class="one">This is a test.</div>
</div>
```
```css
html{
    font-size: 10px;
}
.main{
    width: 100rem;
    height: 80rem;
    font-size: 1.4rem;
    margin: auto;
}
/*媒体查询中仅改变html元素的font-size值即可*/
@media only screen and (max-width: 1000px){
    htnl{font-size: 8px}
}
@media only screen and (max-width: 1200px){
    htnl{font-size: 10px}
}
@media only screen and (max-width: 1500px){
    htnl{font-size: 15px}
}
```

### float布局
>float布局，通过为元素设置float属性，元素将会脱离文档流，并将元素display隐式转换为inline-block，实现多栏布局。但是，float布局存在父元素高度塌陷的问题。

```html
<div class="main">
    <div class="left">This is a test1.</div>
    <div class="right">This is a test2.</div>
</div>
```
```css
/*父元素内所有子元素均为float时高度塌陷*/
/*解决父元素高度塌陷有多种方法*/
.left{
    flaot: left;
}
.right{
    float: right;
}
/*方法一：父元素设置height*/
.main{
    height: 200px;
}
/*方法二：父元素设置为BFC*/
.main{
    overflow: hidden;
}
/*方法三：父元素::after设置clear:both*/
.main::after{
    display: block;
    content: "";
    clear: both;
}

```

### position布局
>position布局，通过为元素设置position属性，absolute/fixed可以使元素脱离文档流。需要注意position+z-index产生的堆叠上下文的层叠顺序。

```html
<div class="super">
    <div class="sub">This is a test.</div>
</div>
```
```css
.super{
    position: relative;
}
.sub{
    position: absolute;
    left: 0;
    top: 100%;
}
```

### flex布局
>flex布局，通过为元素设置display:flex/inline-flex成为container，子元素item根据container的flex-flow/justify-content/align-items等属性进行多栏布局。

```html
<div class="main">
    <div class="item1">111</div>
    <div class="item2">222</div>
</div>
```
```css
.main{
    display: flex;
    flex-direction: row;
    flex-wrap: wrap;
    justify-content: space-around;
    align-items: stretch;
    /*使item内文本超出时换行*/
    word-wrap: break-word;
    word-break: break-all;
}
.item1{
    align-self: center;
}
```
[MDN:CSS_Flexible_Box_Layout](https://developer.mozilla.org/zh-CN/docs/Web/CSS/CSS_Flexible_Box_Layout)

### grid布局
>grid布局，通过为元素设置display:grid/inline-grid成为container，子元素item根据container的grid-template-columns/grid-template-rows等属性进行布局。

```html
<div class="main">
    <div class="item1">111</div>
    <div class="item2">222</div>
    <div class="item3">333</div>
</div>
```
```css
.main{
    display: grid;
    grid-template-columns: 1fr 2fr 1fr;
    grid-gap: 10px;
    grid-template-rows: 200px;
}
```
[MDN:CSS_Grid_Layout](https://developer.mozilla.org/zh-CN/docs/Web/CSS/CSS_Grid_Layout)

## 块和堆叠上下文
### 块格式化上下文
**块格式化上下文（BFC）**

BFC元素及其内部元素不会受到外部元素影响也不会影响外部元素，形成了封闭的块级上下文环境。

```html
<div class="wrap">
    <div class="left">This is a BFC.</div>
    <p>This is a message.</p>
</div>
```
```css
.wrap{
	width: 200px;
    overflow: auto;
}
.left{
    float: left;
}
```
创建块格式化上下文：
- 根元素（html）
- 设置display：inline-block/flow-root
- 设置float，且不等于none
- 设置overflow，且不等于visible
- 设置position：absolute/fixed
- 设置contain：layout/content/paint
- flex布局，display：flex/inline-flex
- grid布局，display：grid/inline-grid
- 表格单元格/标题，display：table-cell/table-caption
- 匿名表格单元格元素：display：table/table-row……
- 多列容器，column-count/column-width/column-span

块格式化上下文特点：
- BFC元素margin不会与其他元素的margin重叠
- BFC元素可以去除float影响，解决父元素高度塌陷问题
- BFC可以实现更加智能的自适应布局

[MDN:Block_formatting_context](https://developer.mozilla.org/zh-CN/docs/Web/Guide/CSS/Block_formatting_context)

### 堆叠层级上下文
**堆叠上下文**

结合z-index使用，决定了一个堆叠上下文中元素在z轴上的显示顺序。

**元素层叠顺序**
backgroun/border`<`负z-index`<`block盒子`<`float盒子`<`inline盒子`<`z-index：auto/0`<`正z-index

```html
<div class="stack">stacking context</div>
<div class="normal">normal context</div>
```
```css
.stack{
	position: absolute;
	z-index: 1; 
}
.normal{
	width: 100px;
}
```



形成堆叠上下文：

- 根元素（html）
- 设置position：relative/absolute/fixed/sticky
- 设置contain：layout/paint……
- 设置isolation：isolate
- 设置opacity，且属性值小于1
- 设置max-blend-mode，且值不为normal
- 设置transform/filter/perspective/clip-path/mask/mask-image/mask-border，且值不为none
- flex布局的子元素，且z-index不为auto
- grid布局的子元素，且z-index不为auto

堆叠上下文特点：
- 层叠上下文可以包含在其他层叠上下文中，并且一起创建一个层叠上下文的层级
- 每个层叠上下文都完全独立于它的兄弟元素：当处理层叠时只考虑子元素
- 每个层叠上下文都是自包含的：当一个元素的内容发生层叠后，该元素将被作为整体在父级层叠上下文中按顺序进行层叠

[MDN:The_stacking_context](https://developer.mozilla.org/zh-CN/docs/Web/Guide/CSS/Understanding_z_index/The_stacking_context)
